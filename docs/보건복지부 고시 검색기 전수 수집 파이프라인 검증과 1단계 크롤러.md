## 수집 대상과 공식 출처

본 과제의 “고시 전수 수집”은 **entity ["organization","보건복지부","south korea ministry"] 웹사이트의 ‘훈령/예규/고시/지침’ 게시판**을 1차 원천으로 삼고, 이후 **entity["organization","국가법령정보센터","law.go.kr portal"](법제처 서비스)**를 2차 정규화 원천으로 삼는 방식이 가장 현실적입니다. 게시판 자체가 목록·상세·첨부의 구조를 이미 갖고 있고, 게시판 화면에 “행정규칙 전문은 국가법령정보센터에서 확인 가능”이라는 안내가 명시되어 있어 “게시판→국가법령정보센터”의 분업이 사실상 공식 흐름에 가깝습니다. citeturn19view0

게시판 목록 화면에는 **제·개정(제정/일부개정/전부개정/폐지) 필터**와 **법령구분(훈령/예규/고시/지침) 필터**가 존재하며, 목록 표의 기본 컬럼이 “번호/구분/발령번호/제목/등록일/미리보기”로 안내됩니다. citeturn20view0  
또한 목록에는 실제로 “구분=일부개정”, “발령번호=보건복지부 고시 제2025-114호”, “제목에 [고시] 태그”가 함께 노출되는 형태의 레코드가 반복적으로 나타납니다. citeturn20view0

전수 수집 규모는 시점에 따라 변하지만, 예시로 2026년 3월 15일 현재 접근 가능한 목록 페이지에서는 **“전체 7,383건”**, **“페이지 30/739”** 같은 형태로 총량이 표시됩니다(즉, 하드코딩이 아니라 런타임에 페이지 수를 파싱해야 합니다). citeturn20view0

운영 단계의 증분 수집은 RSS가 가장 싸고 안정적입니다. 정보구독 페이지에 “훈령/예규/고시/지침” RSS 주소가 별도로 제공됩니다. citeturn9view0

## 목록 전수 수집에서 확보 가능한 필드와 함정

목록 단계에서 “고시”만 대상으로 삼는다면, **페이지를 1부터 끝까지 순회하되, 각 행에서 제목의 `[고시]` 표식을 기준으로 필터링**하는 방식이 구현 리스크가 가장 낮습니다. 실제 목록은 제목에 `[고시]`가 포함되는 패턴이 반복적으로 나타납니다. citeturn20view0turn19view1

목록 수집으로 확보 가능한 핵심 필드는 다음과 같습니다.

- **제·개정 구분**: 목록의 “구분” 컬럼에 “제정/일부개정 …”이 노출됩니다. citeturn20view0  
- **발령번호(원문 문자열)**: 발령번호 컬럼이 “보건복지부 고시 제2025 - 114호”, “제2025-113호”, “고시아님”처럼 **형태가 섞여** 들어옵니다. 따라서 처음에는 `notice_no_raw`로 그대로 저장하고, 정규화(`notice_no_norm`)는 후단에서 해야 합니다. citeturn20view0  
- **제목/등록일/미리보기(요약)**: 목록에 “미리보기” 텍스트가 실제로 함께 노출되어 초기 검색 품질에 도움이 됩니다. citeturn20view0turn19view1  
- **상세 URL**: 제목 링크로 각 게시물 상세 페이지로 이동합니다. citeturn20view0turn21view0  

주의할 점은, 과거 데이터(오래된 페이지)에서는 “구분(제·개정)”이 비어 보이거나 표기 규칙이 달라서 보일 수 있다는 점입니다. 그러므로 “구분이 비어도 레코드로 저장”하고, 상세 단계에서 메타데이터로 보강하는 편이 안전합니다. citeturn19view1

## 목록 전수 수집 파이썬 스크립트

아래 스크립트는 **(1)列表 페이지 수 자동 파싱 → (2) page=1..N 순회 → (3) `[고시]`만 필터링 → (4) SQLite에 upsert 저장**까지 수행하는 1단계 구현입니다. 목록 페이지가 “전체 7,383건 / 30/739”처럼 페이지 수를 텍스트로 제공하므로, 이를 정규식으로 읽어 end_page를 자동 설정합니다. citeturn20view0

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
MOHW(보건복지부) '훈령/예규/고시/지침' 게시판(bid=0026)에서 [고시] 목록을 전수 수집하여 SQLite에 저장한다.

1단계(목록 수집):
- nPage=1..N 순회
- 각 행에서 (제·개정 구분, 발령번호, 제목, 등록일, 미리보기, 상세URL) 추출
- 제목의 [고시] 태그로 필터링하여 DB 적재

설치:
  pip install requests beautifulsoup4 lxml

실행 예:
  python mohw_gosi_list_crawl.py --db mohw_gosi.db
  python mohw_gosi_list_crawl.py --start-page 1 --end-page 50
"""

import argparse
import datetime as dt
import logging
import re
import sqlite3
import time
from typing import Dict, List, Optional, Tuple
from urllib.parse import parse_qs, urljoin, urlparse

import requests
from bs4 import BeautifulSoup


BASE = "https://www.mohw.go.kr"
LIST_ENDPOINT = "/board.es"

DEFAULT_BID = "0026"
DEFAULT_MID = "a10409020000"
DOC_TYPES = ("훈령", "예규", "고시", "지침")


def build_list_url(n_page: int, bid: str, mid: str) -> str:
    return f"{BASE}{LIST_ENDPOINT}?bid={bid}&mid={mid}&nPage={n_page}"


def normalize_ws(s: str) -> str:
    return re.sub(r"\s+", " ", s).strip()


def extract_total_pages(page_text: str) -> Optional[int]:
    # 예: "페이지 30/739"
    m = re.search(r"페이지\s*\d+\s*/\s*(\d+)", page_text)
    return int(m.group(1)) if m else None


def extract_total_count(page_text: str) -> Optional[int]:
    # 예: "전체 7,383건"
    m = re.search(r"전체\s*([\d,]+)\s*건", page_text)
    return int(m.group(1).replace(",", "")) if m else None


def parse_doc_type_from_title(title: str) -> str:
    m = re.search(r"\[(훈령|예규|고시|지침)\]", title)
    return m.group(1) if m else ""


def find_list_table(soup: BeautifulSoup) -> Tuple[Optional[BeautifulSoup], List[str]]:
    """
    목록 테이블을 찾고, 헤더(th) 텍스트를 반환한다.
    페이지 구조가 바뀌면 이 함수부터 조정하면 된다.
    """
    for table in soup.find_all("table"):
        headers = [normalize_ws(th.get_text(" ", strip=True)) for th in table.select("thead th")]
        if not headers:
            continue
        # 보건복지부 해당 목록은 보통 '번호/제목/등록일'이 존재한다.
        if ("번호" in headers) and ("제목" in headers) and (("등록일" in headers) or ("작성일" in headers)):
            return table, headers

    # fallback: 첫 번째 table
    table = soup.find("table")
    headers = [normalize_ws(th.get_text(" ", strip=True)) for th in table.select("thead th")] if table else []
    return table, headers


def map_by_headers(headers: List[str], cells: List[str]) -> Dict[str, str]:
    row = {}
    for idx, h in enumerate(headers):
        row[h] = cells[idx] if idx < len(cells) else ""
    return row


def parse_list_page(html: str, list_url: str) -> Tuple[List[Dict[str, str]], Dict[str, Optional[int]]]:
    soup = BeautifulSoup(html, "lxml")

    page_text = normalize_ws(soup.get_text(" ", strip=True))
    meta = {
        "total_pages": extract_total_pages(page_text),
        "total_count": extract_total_count(page_text),
    }

    table, headers = find_list_table(soup)
    if not table:
        raise RuntimeError("목록 테이블을 찾지 못했습니다. 페이지 구조 변경 가능성이 큽니다.")

    tbody = table.find("tbody")
    trs = tbody.find_all("tr") if tbody else table.find_all("tr")

    items: List[Dict[str, str]] = []
    pending: Optional[Dict[str, str]] = None

    for tr in trs:
        # 본문(메인) 행 판별: 상세 링크(act=view)가 존재하는 행
        a = tr.find("a", href=re.compile(r"act=view"))
        tds = tr.find_all("td", recursive=False) or tr.find_all("td")

        if a and tds:
            cells = [normalize_ws(td.get_text(" ", strip=True)) for td in tds]
            row = map_by_headers(headers, cells)

            title = normalize_ws(a.get_text(" ", strip=True))
            detail_url = urljoin(BASE, a.get("href", ""))

            # list_no 파싱 (상세 URL의 핵심 키)
            qs = parse_qs(urlparse(detail_url).query)
            list_no = qs.get("list_no", [""])[0]

            item = {
                "board_no": row.get("번호", ""),  # 목록표의 '번호'(정렬/표시용)
                "revision_type": row.get("구분", ""),  # 제정/일부개정/전부개정/폐지 등
                "notice_no_raw": row.get("발령번호", ""),  # 그대로 저장 (정규화는 후단)
                "title": title,
                "doc_type": parse_doc_type_from_title(title),
                "posted_date": row.get("등록일", row.get("작성일", "")),
                "preview": "",
                "detail_url": detail_url,
                "list_url": list_url,
                "list_no": list_no,
            }
            items.append(item)
            pending = item
            continue

        # 미리보기(요약) 행: 보통 td 1개(colspan) + 링크 없음
        if pending and (len(tds) == 1) and (tr.find("a") is None):
            pending["preview"] = normalize_ws(tds[0].get_text(" ", strip=True))
            pending = None

    return items, meta


def init_db(conn: sqlite3.Connection) -> None:
    conn.execute(
        """
        CREATE TABLE IF NOT EXISTS notice_list(
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            bid TEXT NOT NULL,
            mid TEXT NOT NULL,
            npage INTEGER NOT NULL,
            list_no TEXT NOT NULL,

            board_no TEXT,
            title TEXT NOT NULL,
            doc_type TEXT,
            revision_type TEXT,
            notice_no_raw TEXT,
            posted_date TEXT,
            preview TEXT,

            detail_url TEXT NOT NULL,
            list_url TEXT NOT NULL,

            crawled_at TEXT NOT NULL,
            UNIQUE(bid, list_no)
        );
        """
    )
    conn.execute("CREATE INDEX IF NOT EXISTS idx_notice_list_doc_type ON notice_list(doc_type);")
    conn.execute("CREATE INDEX IF NOT EXISTS idx_notice_list_posted_date ON notice_list(posted_date);")
    conn.commit()


def upsert_notice_list(
    conn: sqlite3.Connection,
    row: Dict[str, str],
    bid: str,
    mid: str,
    npage: int,
    crawled_at: str,
) -> None:
    conn.execute(
        """
        INSERT INTO notice_list(
            bid, mid, npage, list_no,
            board_no, title, doc_type, revision_type, notice_no_raw, posted_date, preview,
            detail_url, list_url,
            crawled_at
        )
        VALUES(
            :bid, :mid, :npage, :list_no,
            :board_no, :title, :doc_type, :revision_type, :notice_no_raw, :posted_date, :preview,
            :detail_url, :list_url,
            :crawled_at
        )
        ON CONFLICT(bid, list_no) DO UPDATE SET
            npage=excluded.npage,
            board_no=excluded.board_no,
            title=excluded.title,
            doc_type=excluded.doc_type,
            revision_type=excluded.revision_type,
            notice_no_raw=excluded.notice_no_raw,
            posted_date=excluded.posted_date,
            preview=excluded.preview,
            detail_url=excluded.detail_url,
            list_url=excluded.list_url,
            crawled_at=excluded.crawled_at;
        """,
        {
            **row,
            "bid": bid,
            "mid": mid,
            "npage": npage,
            "crawled_at": crawled_at,
        },
    )


def fetch_html(session: requests.Session, url: str, timeout: int, retries: int = 3) -> str:
    last_exc: Optional[Exception] = None
    for attempt in range(1, retries + 1):
        try:
            resp = session.get(url, timeout=timeout)
            resp.raise_for_status()
            # requests가 인코딩을 잘못 잡는 경우가 있어 최소 방어 로직
            resp.encoding = resp.encoding or "utf-8"
            return resp.text
        except Exception as e:
            last_exc = e
            time.sleep(1.2 * attempt)
    raise RuntimeError(f"GET 실패: {url}") from last_exc


def main() -> None:
    ap = argparse.ArgumentParser()
    ap.add_argument("--db", default="mohw_gosi.db")
    ap.add_argument("--bid", default=DEFAULT_BID)
    ap.add_argument("--mid", default=DEFAULT_MID)

    ap.add_argument("--start-page", type=int, default=1)
    ap.add_argument(
        "--end-page",
        type=int,
        default=0,
        help="0이면 start-page 페이지에서 total_pages를 파싱해 자동 설정",
    )
    ap.add_argument("--sleep", type=float, default=0.3, help="요청 간 대기(초)")
    ap.add_argument("--timeout", type=int, default=20)
    ap.add_argument("--target-doc-type", default="고시", choices=list(DOC_TYPES))
    ap.add_argument("--log", default="INFO")

    args = ap.parse_args()

    logging.basicConfig(
        level=getattr(logging, args.log.upper(), logging.INFO),
        format="%(asctime)s %(levelname)s %(message)s",
    )
    log = logging.getLogger("mohw_gosi_list")

    conn = sqlite3.connect(args.db)
    init_db(conn)

    session = requests.Session()
    session.headers.update(
        {
            "User-Agent": "Mozilla/5.0 (compatible; mohW-gosi-crawler/1.0)",
            "Accept-Language": "ko-KR,ko;q=0.9,en;q=0.7",
        }
    )

    # 1) start-page를 먼저 읽어서 total_pages를 파싱
    first_url = build_list_url(args.start_page, args.bid, args.mid)
    first_html = fetch_html(session, first_url, timeout=args.timeout)
    _, meta = parse_list_page(first_html, first_url)

    total_pages = meta.get("total_pages")
    total_count = meta.get("total_count")
    if total_count is not None:
        log.info("페이지 텍스트 기준 총 건수: %s", total_count)
    if total_pages is not None:
        log.info("페이지 텍스트 기준 총 페이지: %s", total_pages)

    # 2) end_page 결정
    end_page = args.end_page if args.end_page and args.end_page > 0 else (total_pages or args.start_page)
    if end_page < args.start_page:
        raise ValueError("end-page는 start-page 이상이어야 합니다.")

    # 3) 본 수집 루프
    total_upserts = 0

    for npage in range(args.start_page, end_page + 1):
        list_url = build_list_url(npage, args.bid, args.mid)
        html = fetch_html(session, list_url, timeout=args.timeout)
        items, _ = parse_list_page(html, list_url)

        page_upserts = 0
        crawled_at = dt.datetime.now(dt.timezone.utc).isoformat()

        for item in items:
            # 스코프: [고시]만 적재
            if item.get("doc_type") != args.target_doc_type:
                continue

            # list_no가 비면 후속 단계(상세/첨부) 연결 키가 사라지므로 일단 스킵
            if not item.get("list_no"):
                continue

            upsert_notice_list(conn, item, bid=args.bid, mid=args.mid, npage=npage, crawled_at=crawled_at)
            page_upserts += 1

        conn.commit()
        total_upserts += page_upserts
        log.info("page %s/%s: parsed_rows=%s, upserted_gosi=%s", npage, end_page, len(items), page_upserts)

        time.sleep(args.sleep)

    log.info("DONE. total_upserts=%s (target=%s)", total_upserts, args.target_doc_type)


if __name__ == "__main__":
    main()
```

### 스크립트가 “1단계 요구사항”을 충족하는 방식
- **페이지 1→끝 페이지 순회**는 `nPage` 파라미터와 목록 화면의 “페이지 x/y” 표기를 사용해 구현했습니다. citeturn20view0turn19view0  
- **저장 필드**는 목록 화면이 제공하는 “구분/발령번호/제목/등록일/미리보기”를 그대로 가져오고, 상세 URL을 함께 저장합니다. citeturn20view0turn19view1  
- **고시만 수집**은 제목의 `[고시]` 표식을 기준으로 필터링합니다. citeturn20view0turn19view1  

## 상세·첨부 메타데이터 수집 단계로 확장하는 방법

목록만으로도 “제목/발령번호 기반 검색”은 빠르게 만들 수 있지만, 실무형 검색기는 **상세 메타데이터 + 첨부(별표·별지 포함) 텍스트**가 사실상 본체입니다. 실제 상세 페이지에는 작성일(시간 포함), 담당부서, 제·개정 구분, 분류(고시), 제·개정일, 발령번호가 구조화되어 노출됩니다. citeturn22view0turn8view0

또한 상세 페이지의 ‘첨부파일’ 영역은 최소 2개(예: HWPX+PDF)부터 여러 개(전문/별표 분리 등)까지 흔하며, 형식도 PDF/HWPX/HWP뿐 아니라 XLSX까지 섞여 나옵니다. citeturn22view0turn8view1turn5search6

따라서 단계는 다음처럼 **의도적으로 분리**하는 것이 맞습니다.

- **상세 수집기(메타데이터+본문 요약)**: 상세 페이지 HTML을 저장(`raw_html`)하고, 화면에 노출된 메타 필드(제·개정일, 발령번호 등)를 파싱해 `notice_detail`에 적재합니다. citeturn22view0turn8view0  
- **첨부 메타 수집기(파일명+URL만)**: 첨부파일의 “다운로드 링크/파일명/확장자/크기”를 `attachment` 테이블에 저장하되, 이 단계에서는 파일다운로드를 하지 않습니다. citeturn22view0turn8view1turn5search6  
- **첨부 텍스트 추출기(2차 작업)**: PDF부터 텍스트를 추출해 검색 인덱스에 포함하고, HWP/HWPX는 후순위로 처리합니다(복지부 고시는 PDF를 함께 제공하는 경우가 적지 않음). citeturn22view0turn8view1  

여기서 개인정보 최소화 원칙을 적용하면, 상세 페이지에 “담당자/연락처”가 노출되긴 하지만 검색 품질과 직접 상관이 적으므로 **저장(또는 인덱싱)을 생략**하고 “담당부서”만 저장해도 대부분의 사용 시나리오를 충족할 수 있습니다. citeturn22view0turn8view0

## 법제처 매핑 전략

게시판 자체가 “행정규칙 전문은 법제처 서비스인 국가법령정보센터에서 확인”이라고 명시하므로, 장기적으로는 **게시판 레코드 ↔ 행정규칙(고시) 레코드 매핑**이 검색기의 지속가능성을 크게 올립니다. citeturn19view0

행정규칙 매핑은 다음 조합이 정석입니다.

- 목록 조회: `lawSearch.do?target=admrul`  
- 본문 조회: `lawService.do?target=admrul` citeturn13view0turn13view1  
- 고시 필터: `knd=3` (행정규칙 종류: 1=훈령, 2=예규, 3=고시 …) citeturn13view0  
- 소관부처 필터: `org=<부처코드>` citeturn13view0  

여기서 핵심은 “보건복지부 소관부처코드”인데, 국가법령정보 공동활용의 코드 테이블(소관부처코드 다운로드)에서 **보건복지부=1352000**으로 확인됩니다. citeturn17view0

실무 매칭 로직은 보통 아래 우선순위를 탑니다.

- 1순위: **발령번호 기반 매칭**  
  - 목록의 `notice_no_raw`에서 숫자만 추출해 `nb` 파라미터 후보를 만들고(문서 예시상 “제2023-8호 → nb=20238” 같은 규칙 제시), 결과가 1건이면 매칭 확정. citeturn13view0turn22view0  
- 2순위: **제목(행정규칙명) 기반 매칭**  
  - `query=<정확한 고시명>`으로 검색(`search` 기본=행정규칙명) 후, 결과 다건이면 유사도(완전일치/부분일치/정규화 일치)로 점수화하여 `matched_score` 기록. citeturn13view0  
- 3순위: **발령일/시행일 보조 조건**  
  - 상세 메타데이터에서 `제.개정일`을 파싱해 law.go.kr 결과의 발령일자/시행일자와 교차검증(동명이 고시가 있을 수 있음). citeturn13view0turn22view0  

참고로 법제처 API는 **공공데이터포털에서도 “LINK 타입” 오픈API로 노출되는 경우가 있으며**, 예: 행정규칙 신구법 비교 목록 조회가 data.go.kr에 등록되어 있고 실제 가이드는 open.law.go.kr로 연결됩니다. 이미 data.go.kr 기반 인증/키 관리가 익숙하다면 이 경로도 운영 편의상 선택지가 될 수 있습니다. citeturn18view0turn14search6

## DB 스키마·검색 인덱스·증분 갱신 운영

사용자가 고시를 찾는 방식은 “고시번호(발령번호)”, “제목 키워드”, “본문 문장”, “별표·별지 용어”가 핵심이므로, 설계는 결국 **텍스트 인덱싱 범위를 어디까지 포함하느냐(첨부 포함 여부)**에 의해 성패가 갈립니다. 첨부파일이 다수이고 형식이 다양하다는 점(특히 PDF/HWPX/XLSX 혼재)은 이 인덱싱 설계를 ‘분리 파이프라인’으로 만들 이유를 뒷받침합니다. citeturn8view1turn5search6turn22view0

초기부터 최소 4테이블 분리는 타당합니다.

- `notice_list`: 목록에서 온 “찾기용” 메타  
- `notice_detail`: 상세 페이지에서 온 “정확한” 메타 + 본문 + raw_html  
- `attachment`: 첨부 메타 + 로컬 경로 + 추출 텍스트  
- `law_mapping`: 법제처 매칭 결과(식별자/링크/점수)

이 구조는 실제 사이트 구조가 “목록→상세→첨부”로 분리되어 있고, 상세에 조직/일자/발령번호/첨부가 함께 노출되기 때문에 특히 적합합니다. citeturn22view0turn20view0

운영(증분 갱신)은 RSS를 기준으로 잡는 게 맞습니다. 보건복지부 정보구독 페이지가 “훈령/예규/고시/지침” RSS 주소를 공식적으로 제공하므로, 하루 2~4회 폴링 구조로 신규 게시물만 상세/첨부 파이프라인에 태우면 “전체 재수집”을 피할 수 있습니다. citeturn9view0
